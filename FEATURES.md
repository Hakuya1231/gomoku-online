# 五子棋项目功能点清单

本文档与当前代码（`app.js`、`network.js`、`index.html` 等）对齐，用于重构前统一预期。若实现变更，请同步更新本文档。

---

## 1. 游戏核心（本地棋盘）

| 功能点 | 说明 |
|--------|------|
| 棋盘尺寸 | 支持 15×15、19×19；切换后重开；星位随尺寸变化 |
| 落子与回合 | 黑棋先手；轮流落子；连成五子判胜并高亮胜利线 |
| 胜负判定 | 横、纵、主斜、反斜四向；恰好五连 |
| 不可重复落子 | 已有子的交点忽略落子 |
| 终局后禁止落子 | 已有胜者时不再接受落子 |
| 重开 | 按钮；联机模式下仅主机可重开并下发重置消息 |
| 悔棋 | **不提供悔棋功能**（重构时将删除现有悔棋相关代码与网络消息） |
---

## 2. 规则与设置

| 功能点 | 说明 |
|--------|------|
| 禁手（三三 / 四四 / 长连） | 仅对**黑棋**生效；需开启「禁手规则」开关（默认开启） |
| 五连优先于禁手 | 落子后若形成恰好五连获胜，不判禁手；长连（六子及以上）按禁手处理 |
| 坐标显示 | 开关控制棋盘四周坐标 |
| 悬停提示 | 开关控制空位悬停提示线；禁手位在黑棋回合且开启禁手时显示红叉与类型文案 |
| 「你执」黑 / 白 | 决定本地玩家执子颜色；**先手始终为黑棋**（与仅选执子不混为一谈） |
| 人机难度 | 简单（启发式随机）、中等（搜索深度 4）、困难（深度 6） |
| 对战模式 | 本地双人（PVP）、人机（AI）、在线联机（ONLINE） |

**文档注意**：`README.md` 中若写「黑先/白先可选」，与代码「永远黑先」不一致时，以代码为准，建议 README 改为「黑棋先手；可选择你执黑或执白」。

## 3. 渲染与交互

| 功能点 | 说明 |
|--------|------|
| Canvas 棋盘 | 网格、渐变背景、天元与星位 |
| 棋子绘制 | 黑白棋样式、阴影、最后一手标记 |
| 胜利连线 | 高亮显示 |
| 点击映射 | 交点附近阈值约 0.35 格宽，映射到最近交叉点 |
| 状态栏 | 当前回合、胜负、联机角色与连接状态等 |
| 副标题 | 模式文案 + 棋盘尺寸（如 `本地双人对战 · 15×15`） |
| 对局信息面板 | 步数、上一步、胜者 |

---

## 4. 人机（AI）

| 功能点 | 说明 |
|--------|------|
| AI 自动应子 | 轮到 AI 时自动落子；短延迟；`aiThinking` 期间更新文案 |
| 回合约束 | 人类无法在 AI 回合点击落子 |
| 禁手与 AI | AI 执黑也遵守禁手，需在 AI 落子路径单独与 `place` 对齐 |

---

## 5. 联机（Firebase Realtime Database）

| 功能点 | 说明 |
|--------|------|
| 创建房间 | 生成房间号与分享链接 |
| 通过链接加入 | URL 参数 `room`（及解析嵌套 URL） |
| 角色 | 主机、客机、观战（房间已满时第三人） |
| 配置同步 | 棋盘大小、禁手开关、主机执子选择 |
| 落子同步 | 通过消息队列推送，对端应用局面 |
| 重开同步 | 仅主机发起重置 |
| 观战 | 请求全量棋盘状态；主机可推送 `board_state` |
| 断开与清理 | 断开按钮；主机离开可删除房间等（见 `network.js`）；断开后界面保持联机模式，可再次创建新的房间 |
| 未连接保护 | 未连接时落子会提示，不修改局面 |

**实现备注**：`place` 在 `placeInternal` 之后调用 `sendMove(r, c, state.turn)`，此时回合可能已切换；若协议消费方依赖「刚落子一方」的标识，需与当前 `state.turn` 语义核对或修正。

## 6. 测试与代码一致性（非功能，但对质量有关）

| 项目 | 说明 |
|------|------|
| `test.js`（Node） | 覆盖胜负、禁手、边界、19×19 等；`checkWinFrom` 与 `app.js` 在「五连线段截取」上可能略有差异（测试取前 5 点，主程序用 `clamp` 取含最后落子的五连段） |
| `test.html` | 浏览器内自测；部分逻辑与 `test.js` / 主程序不完全一致，不宜作为唯一真理来源 |
| `e2e.spec.js` | Playwright；依赖 `place` 全局或模拟点击；覆盖 UI、胜负、禁手开关、AI、19×19、部分联机 |
| `TEST_CASES.md` / `TEST_PROGRESS.md` | 用例表与执行记录；部分条目与自动化覆盖、代码演进可能不同步，以实际测试为准 |
